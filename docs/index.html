Local browser automation over MCP

One browser stack. Four client paths. No remote browser farm.

BrowseFleetMCP pairs a local stdio MCP server with a Chrome extension so Codex, Cursor, Claude, and other MCP clients can drive your real browser profile with isolated sessions.

Setup in four moves

Server Extension Client config
  1. 01 Build the root MCP server.
  2. 02 Build and load the Chrome extension.
  3. 03 Add the stdio server to your MCP client.
  4. 04 Connect a Chrome tab from the extension popup.

The client config alone is not enough. If no tab is connected, tool calls will fail with No connected tab.

What you install

Server, extension, then client config.

Root package runs the stdio MCP server.

Chrome extension bridges tabs into leased sessions.

Client config launches the server from Codex, Cursor, Claude, or any stdio-capable MCP host.

Install Flow

Build once, then pick your client.

1

Build the MCP server

Do this at the repo root.

npm install
npm run build
2

Build the Chrome extension

Then load extension-v2/ in chrome://extensions.

cd extension-v2
npm install
npm run build
3

Add the server to your client

Use the published package or point directly at dist/index.js.

{
  "command": "npx",
  "args": ["-y", "browsefleetmcp"]
}
4

Connect a tab

Open the extension popup in Chrome and connect the current tab before using browser tools.

Client Snippets

Pick the MCP client you actually use.

Codex

Codex supports both CLI setup and direct ~/.codex/config.toml edits. The same config is shared across the CLI and IDE integration.

Repo examples 
codex mcp add browsefleet -- npx -y browsefleetmcp

Cursor

Put the config in ~/.cursor/mcp.json, then restart Cursor.

Repo examples 
{
  "mcpServers": {
    "browsefleet": {
      "command": "npx",
      "args": ["-y", "browsefleetmcp"]
    }
  }
}

Claude Code

Use the CLI for local or project-scoped setup. If you want the config in version control, add --scope project.

Repo examples 
claude mcp add --transport stdio browsefleet -- npx -y browsefleetmcp

Claude Desktop

Claude Desktop now prefers a local MCP bundle. This repo includes the root manifest.json and .mcpbignore needed to package one.

Open manifest 
npm install -g @anthropic-ai/mcpb
npm run build
mcpb pack

Generic stdio clients

Any MCP client that can launch a local command with command and args should work with the same server entrypoint.

Repo examples 
{
  "command": "npx",
  "args": ["-y", "browsefleetmcp"]
}

Known gotchas

If tools connect but browser actions fail, it is almost always the extension state.

Build the root server before pointing any client at dist/index.js.

Reload the extension after rebuilding extension-v2/.

Reconnect a tab from the popup if you hit No connected tab.

On native Windows, wrap npx with cmd /c if the client cannot spawn it directly.
